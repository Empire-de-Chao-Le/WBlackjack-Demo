in the Language Tab there is a button "Take a look" for every language. currently it just displays the list of word entries (Word Pool) for this language, with search - viewing mode. Let's add selection mode and editing functionality to this list. 

- add a "Select" button to the list (at the top right corner, the level of "Back" button) 
- when the "Select" button is clicked, the list goes into selection mode: becomes selectable. the selection is made by clicking on the entry line, it gets highlighted. the button "Select" can be clicked again to go back from the selection mode to viewing mode (in this case, the selection is cleared and no action performed on the list). 
- when the "Select" button is clicked, and the listgoes into selection mode, a bottom bar appears withe the buttons "clear selection" and "delete selected" and space for naming selected items (the bottom bar should stick to the bottom of the screen to remain visible with scrolling; the bottom bar appears in selection mode and disapperas in viewing mode) 
- when at least one entry line is selected, the buttons "clear selection" and "delete selected" become active. 
- important: search can be used in the selection mode, the results are selectable, and once clicked, the selection REMAINS even when the search is cleared or changed to a new input. Selection state must persist across search queries. once made, selection can be removed only either by clicking on the selected entry again (de-selects this particlular netry), by clicking "clear selection" (de-selects all selected entries) or by swithing off selection mode via "Select" button.  
- to keep the track of the selected items, in the bottom strip there is a simple list of words from all selected entries (only "word" field is used, without the "translation" field!), comma separated, smal font, two lines only, truncate the rest to "...and X more". 
- clicking "clear selection" button in the bottom strip clears selection, but keeps selection mode. 
- clicking "Select" button at the top strip when there are some entries already selected: clears selection and returns from selection mode to viewing mode (does NOT delete any entries)
- clicking "delete selected": first displays a confirmation dialogue with the FULL list of the entries to be deleted (options No/Yes. the "No" option takes back to the list with selection still intact; basically, just closes the confirmation dialogue without any actions on the entries. 'Yes' triggers the deletion.). If the option "Yes" is clicked by the user, the entries from the Word Pool are deleted. the entries must be deleted from the database as if they never existed, with no remaining calls from flashcards progress etc. suggest to first delete any related records in flashcard_progress (and any other tables that reference word_pool.id) to avoid Foreign Key constraint errors as the first step, then delete from the word_pool table. make sure deleting a word will not result in any errors in flashcards, there will be no calls to non-existing entries. 
- after the deletion is confirmed by the user and performed by the app, the app takes the user back to the Word Pool, in viewing mode. 
note that deleting words from the Word Pool should not affect song-specific Exercises in any way, as the song-specific lessons use CSV file data. also, the possibility of adding the same entry again remains, which is fine. 
the "Ignore" option and behaviour in the Flashcards and Word Pools remains intact.
